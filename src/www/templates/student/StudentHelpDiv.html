Student Help Page

Greetings, student! Welcome to UTDesign GettIt, the fancy, new, and very sexy procurement system for UTDesign Projects. Allow me to acquaint you with it!

The goal of this system is to help you, the student, to submit procurement requests so that they may be approved in a timely and orderly fashion.

You will notice at the top of the interface a “Logout” button. Clicking on the logout button will log you out of the system.

You will also notice four (4) tabs. You’re currently viewing the “About/Help” tab. The other three (3) are:

“Request Dashboard”
“Create Request”
“Budget”

Let’s discuss each of them.

Request Dashboard

The “Request Dashboard” view allows you to view a list of all requests you’ve previously created, including requests that have been submitted, saved but not submitted, and cancelled.

By default, you will see a table of requests that displays:
“Request Number”: a unique integer that differentiates your request from every other request in the system, including unsubmitted ones.
“Project Number”: the project number of the project this request is for; If you are working on only one project, this value will be the same for all of your requests.
“Status”: the status of your request
“Vendor”: name of vendor the items will be ordered from. Each procurement request may have only one vendor
“URL”: URL of the home page of the vendor’s online storefront
“Total Cost”: total cost of all items plus shipping
“Shipping Cost”: the cost of shipping only; this will be set after your items have been ordered

If you click on any of the white rows of this table, the row will expand, revealing gray rows, each corresponding to an item; and optionally buttons, depending on the status of the request.

These buttons may include:
“Clone”: copies the data from this request into the “Create Request” form and allows you to create a new request with it

“Edit”: copies the request into the “Create Request” form and allows you to edit it and save your changes

This action is only possible if the request has not been submitted, or it has been rejected and sent back to you to make changes.

You may stop editing the request by either saving it for later, submitting it, or selecting the “Start New Request” button.

“Cancel”: allows you to cancel a request. After you cancel a request, it may no longer be edited or submitted.

This action is only possible if the request has not been submitted, or it has been rejected and sent back to you to make changes.

“View History”: brings up a table allowing you to view the history of the procurement request

Create Request

The Create Request view allows you to create, save, and submit procurement requests. It gives you a blank form. First, you must fill out each of the empty fields in the form with the required general information for your request.

“Project Number”: which project the request is for; cannot be edited after submission
“Manager Email”: which technical manager your request will be submitted to for approval; cannot be edited after submission
“Vendor”: name of vendor your items will be ordered from. You may specify only one vendor per procurement request
“Vendor URL”: URL of the home page of the vendor’s online storefront
“Total Cost”: total cost of all items plus shipping
“Shipping Cost”: the cost of shipping only; this will be set after your items have been ordered

You may then give information for one or more items. There is no limit to the number of items per request, so long as each item is to be ordered from the same vendor. Each field for each item must be filled in.

“Description”: a brief verbal description of what the item is, e.g. “screws”
“Item URL”: the URL of the item’s online listing
“Catalog Part Number”: the catalog part number of the item
Quantity”: how many of this item you wish to order; please give an integer
“Estimated Unit Cost”: how much one of this item costs
“Total Cost”: automatically figured out for you by the system

The Justification and Additional Info boxes are both optional. Below these boxes, at the bottom of the form, are buttons that perform an action:

The “Submit Request” button submits the request to the system, awaiting approval by the technical manager you specified in the “Manager Email” field. You cannot cancel requests you have submitted to the technical manager.

The “Save Request” button saves the request to the database without submitting it to a technical manager. You will then be able to edit or cancel this request.

The “Start New Request” button discards all data you’ve written to the form. If you did not save the request you were working on before selecting the button, all changes you made will be permanently lost. If you were editing a saved request before selecting this button, you will no longer be doing so.

Budget

The budget interface allows you to review the budget for your project(s), including seeing how much money is remaining of the budget and what expenditures have been made.

At the top of the view is at least one tab. Each tab corresponds to a project. If you’re assigned to only one project, there will be only one tab.

The remainder of the page is separated into three separate tables:

The first table shows the following information:

“Budget Limit”: The total budget of the project
“Available Budget Remaining”: how much money is left after all completed expenditures are taken into account, i.e. only ordered items
“Pending Budget Remaining”: how much money is left after all completed and uncompleted expenditures are taken into account, i.e. both ordered items and unordered items

The second table shows how much has been spent or will be spent per procurement request. Each row of the table is one procurement request.

The following information is shown for each row:

“Status”: the status of the request.
“Vendor”: name of vendor the items will be ordered from. Each procurement request may have only one vendor
“Subtotal”: the total cost of all items in the request
“Shipping”: the total cost of shipping
“Total Cost”: the total cost of all items plus shipping

The third table shows all remaining costs tied to the project. Each row shows information about these costs. These costs come in three types:

“Refund”: money is returned to the project’s budget
“Reimbursement”: money is taken from the project’s budget to reimburse a student who spent their own money on items
“New Budget”: the total budget of the project is set to a new value

Each row of the table shows the following information about each cost:

“Timestamp”: what time and date the cost was applied to the project
“Type”: see above
“Comment”: any comment made by the admin who applied the cost will display here
“Amount”: how much money the cost is. Negative means this cost returns money to the project’s budget.

If the type of the cost is “New Budget”, the “amount” of the cost is the new budget of the project and not the change in the budget. For example, if a project whose budget was $2000 had its budget increased by $1000, the “amount” of the new budget would be $3000.
